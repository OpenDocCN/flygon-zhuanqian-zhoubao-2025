(61赞)学习来自挣扎而非记忆，先做项目再按需搜索信息更有效

作者： 亦仁的收藏夹

日期：2025-10-31

“我们从小被训练的学习模式是: 1. 看教程 2. 记笔记 3. 感觉学会了 4. 实际动手时啥都不会 5. 回去继续看教程(死循环) Dan
Koe的学习哲学极其简单:先做项目，只在卡住时才去搜索信息。 为什么?因为学习来自挣扎，而非记忆。
当你先看教程，你的大脑处于"低需求"状态，信息进去了，转个身就忘了。但当你先做项目，你会暴露自己的知识缺口。这时候你的大脑处
于"高度需求"状态，任何你学到的东西都会被牢牢记住。”


评论区：

诗语 : 看教程这个起手式没错，问题是要回答 看什么教程 看谁的教程 最好是能提升一个meta level问，


曼巴 : 最近正好有一个很深的感悟：很多的教程和收藏，在你真正开始行动之前，对于你来说，也仅仅只是奶头乐罢了


同赴清秋 : 搜索信息，没有那么多精准有用的信息供你搜索的


小趴菜 : 带着问题去学习


琳 : 我最近处于这种状态，想了几天最终开始启动，做的过程当中暴露出很多意想不到的问题，但是也会诞生出很多灵感跟新想法，又会促使我向前一步。如此正向循环。当然结果依然不是非常满意，但是每天都有进步。
