## (精华帖)(53 赞)11 天开发一个业务管理系统？技术小白的 AI 编程实战复盘

作者： 功夫鱼

日期：2025-10-22

周末对账在多张 Excel 里反复核对，让我意识到：与其做表格搬运工，不如让系统替我算。 参加生财 Claude
航海后，我用“ChatGPT 梳任务书，Claude Code 的 plan
模式聊清逻辑，狂暴模式实现”这套方法，在 11 天内做出一个完全贴合我业务的系统：客户/订单/收支自动关联，报价按字数与服务系数并叠加客户等级折扣，坏账自动预警与恢复，仪表盘支持本月/本季/本年与去年同期对比。
结果是：对账从 2 小时缩到 5 秒，遗漏与误算基本归零。技术小白也能做到，前提是你最懂你的业务。 具体内容请移步飞书： [`resp1f21kn.feishu.cn/wiki/Z2lJwnyBiiXUUokzqQ0c5FNwnBb?from=from_copylink`](https://resp1f21kn.feishu.cn/wiki/Z2lJwnyBiiXUUokzqQ0c5FNwnBb?from=from_copylink)

评论区：

Real : 小白请教，是不是原来的方式是从系统导出各种 excel 表去反复核对；
现在的方式是，自己开发了一个网页管理系统，把数据导进去后，根据各种逻辑去比对，分类，清晰展示？

功夫鱼 : 以前没有系统，所以都是自己用 excel 表手动记录，然后自行核对，算相关的数据。
建立系统后，就是根据业务单做好登记，后台抓数据自行计算一段时间内的收入、支出、利润，后台根据业务单记录计算客户成交笔数、金额等。
除此之外，我还做了客户分级和返佣，之前用表格、用微信、考脑子记，现在把业务单登记好之后，系统会根据成交的次数去对客户进行分级，报价的时候按照相应级别打折，给出报价建议，我自己用下来感觉还是挺好的。
